About Student Manager

Project Overview

Student Manager is a comprehensive, modern web application designed to streamline the management of students, courses, enrollments, and academic records in educational institutions.

Technology Stack

Frontend

  • ✓ HTML5
  • ✓ CSS3 with CSS Variables
  • ✓ Vanilla JavaScript (ES6+)
  • ✓ Responsive Design

Backend

  • ✓ Symfony 8.0
  • ✓ PHP 8.4
  • ✓ API Platform
  • ✓ Doctrine ORM

Database

  • ✓ PostgreSQL 16
  • ✓ Relational Design
  • ✓ Data Integrity Constraints
  • ✓ Query Optimization

Architecture

  • ✓ REST API Architecture
  • ✓ Separation of Concerns
  • ✓ Client-Server Model
  • ✓ JSON Data Exchange

Key Features

  • Student Management: Add, update, and manage student information with status tracking
  • Course Management: Create and manage courses with capacity planning and semester organization
  • Enrollment Tracking: Track student course enrollments with status management (enrolled, dropped, completed)
  • Grade Recording: Record grades with automatic letter grade calculation
  • GPA Calculation: Automatic GPA calculation based on recorded grades
  • Responsive Design: Works seamlessly on desktop, tablet, and mobile devices
  • RESTful API: Complete API endpoints for all operations via API Platform
  • Data Validation: Comprehensive input validation and error handling

System Architecture

The Student Manager follows a modern client-server architecture with clear separation of concerns:

  • Frontend Layer: Vanilla HTML5, CSS3, and JavaScript providing a responsive user interface
  • API Layer: RESTful API built with Symfony and API Platform for data operations
  • Business Logic: Repository pattern for data access and service classes for business logic
  • Database Layer: PostgreSQL with Doctrine ORM for data persistence and integrity

Data Model

The system manages five core entities with the following relationships:

  • User: System authentication and authorization (base for Students)
  • Student: Core student records with academic status (extends User)
  • Course: Course catalog with capacity and semester information
  • Enrollment: Junction table linking Students to Courses (many-to-many)
  • Grade: Academic performance records for Students in Courses
